# Claude Usage Dashboard 디자인 명세서

## 개요
- 화면 목적: Claude Code 사용 일지(마크다운)를 파싱하여 종합 통계 대시보드, 일지 목록, 일지 상세를 제공하는 SPA
- 작업 모드: 신규 설계
- 페이지 수: 3개 (홈 대시보드, 일지 목록, 일지 상세)

## 디자인 토큰

프로젝트가 초기 상태이므로 shadcn UI 기본 토큰 체계를 기준으로 한다. Tailwind CSS v3 + shadcn UI의 CSS 변수 기반 테마 시스템을 사용한다.

- 색상 팔레트 (CSS 변수, HSL):
  - `--background`: 라이트 `0 0% 100%` / 다크 `240 10% 3.9%`
  - `--foreground`: 라이트 `240 10% 3.9%` / 다크 `0 0% 98%`
  - `--card`: 라이트 `0 0% 100%` / 다크 `240 10% 3.9%`
  - `--card-foreground`: 라이트 `240 10% 3.9%` / 다크 `0 0% 98%`
  - `--primary`: 라이트 `240 5.9% 10%` / 다크 `0 0% 98%`
  - `--primary-foreground`: 라이트 `0 0% 98%` / 다크 `240 5.9% 10%`
  - `--secondary`: 라이트 `240 4.8% 95.9%` / 다크 `240 3.7% 15.9%`
  - `--muted`: 라이트 `240 4.8% 95.9%` / 다크 `240 3.7% 15.9%`
  - `--muted-foreground`: 라이트 `240 3.8% 46.1%` / 다크 `240 5% 64.9%`
  - `--accent`: 라이트 `240 4.8% 95.9%` / 다크 `240 3.7% 15.9%`
  - `--destructive`: 라이트 `0 84.2% 60.2%` / 다크 `0 62.8% 30.6%`
  - `--border`: 라이트 `240 5.9% 90%` / 다크 `240 3.7% 15.9%`
  - `--ring`: 라이트 `240 5.9% 10%` / 다크 `240 4.9% 83.9%`
  - `--radius`: `0.5rem`
- 차트 전용 색상:
  - `--chart-1`: `220 70% 50%` (파란 계열, 세션 추이선)
  - `--chart-2`: `160 60% 45%` (초록 계열)
  - `--chart-3`: `30 80% 55%` (주황 계열, 도구 호출 추이선 / 바 차트)
  - `--chart-4`: `280 65% 60%` (보라 계열)
  - `--chart-5`: `340 75% 55%` (핑크 계열)
- 간격 체계: Tailwind 기본 4px 단위 (4, 8, 12, 16, 20, 24, 32, 40, 48, 64)
- 타이포그래피:
  - 페이지 타이틀: text-3xl (30px), font-bold, tracking-tight
  - 섹션 타이틀 (카드 헤더): text-lg (18px), font-semibold
  - 통계 값: text-2xl (24px), font-bold
  - 본문: text-sm (14px), font-normal
  - 보조 텍스트: text-sm (14px), text-muted-foreground
  - 캡션/라벨: text-xs (12px), text-muted-foreground
- border-radius: `--radius` 기반 (0.5rem = 8px)
- 아이콘: lucide-react 전용, 기본 크기 16px, 카드 아이콘 20px, 빈 상태 아이콘 64px

### 헤더 상세

- 높이: 64px
- 배경: 라이트 `background` / 다크 `background`
- 하단 경계선: 1px solid `border`
- position: sticky, top: 0, z-index: 50
- 내부 레이아웃: max-width 1280px, 중앙 정렬, 좌우 패딩은 메인 콘텐츠와 동일
- 좌측 영역:
  - 서비스명 텍스트 "Claude Usage Dashboard": text-base, font-bold, 클릭 시 `/`로 이동
  - 네비게이션 링크: "Dashboard", "Daily Logs"
    - 간격: 서비스명과 네비게이션 사이 gap 32px, 링크 간 gap 24px
    - 스타일: text-sm, font-medium, text-muted-foreground
    - 활성 상태: text-foreground, 하단에 2px solid primary 밑줄 (헤더 하단에 맞닿게)
    - 호버: text-foreground 전환
- 우측 영역:
  - 다크 모드 토글 버튼: 아이콘 버튼 (lucide-react Sun / Moon)
  - 크기: 36x36px, border-radius: radius
  - 호버: accent 배경

### 헤더 반응형 (모바일 < 768px)

- 서비스명을 "Claude Usage"로 축약
- 네비게이션 링크는 아이콘으로 대체: Dashboard -> BarChart3, Daily Logs -> CalendarDays

### 섹션 간격

- 통계 카드 ~ 차트 영역: 24px
- 차트 영역 ~ 트렌드 차트: 24px
- 트렌드 차트 ~ 기술 스택: 24px
- 기술 스택 ~ 최근 활동: 24px

### 섹션 1-1: 페이지 타이틀 영역
- 역할: 페이지 식별 및 컨텍스트 제공
- 레이아웃: 단일 컬럼, 좌측 정렬
- 포함 요소:
  - 타이틀 텍스트 "Dashboard": text-3xl, font-bold, tracking-tight
  - 서브타이틀 "클로드 코드 사용 현황 종합": text-sm, text-muted-foreground, 타이틀과 gap 4px

### 섹션 1-2: 통계 카드 그리드
- 역할: 핵심 지표를 한눈에 파악
- 레이아웃: CSS Grid, 4열(데스크탑) / 2열(태블릿) / 1열(모바일), gap 16px
- 카드 공통 스타일:
  - 카드 컨테이너: border 1px solid `border`, border-radius `radius`, padding 24px, 배경 `card`
  - 상단 행: 라벨(text-sm, text-muted-foreground) + 아이콘(우측 정렬, 16px, text-muted-foreground) 가로 배치
  - 값 행: text-2xl, font-bold, 라벨과 gap 8px
  - 보조 텍스트 행: text-xs, text-muted-foreground, 값과 gap 4px

| 순서 | lucide-react 아이콘 | 라벨 | 값 형식 | 보조 텍스트 |
|---|---|---|---|---|
| 1 | Calendar | 활동 일수 | `{N}일` | 전체 기록 기간 |
| 2 | MessageSquare | 총 세션 수 | `{N}개` | 부모 세션 기준 |
| 3 | Wrench | 총 도구 호출 | `{N}회` | 전체 누적 |

```

```

### 섹션 1-3: 차트 영역
- 역할: 도구 사용 분포와 작업 유형 비율을 시각적으로 전달
- 레이아웃: CSS Grid, 2열(데스크탑) / 1열(모바일/태블릿), gap 16px
- 각 차트를 카드 컨테이너로 감싼다

#### 좌측: 도구 사용 현황 (수평 Bar Chart)
- 카드 타이틀: "도구 사용 현황" (Wrench 아이콘 + text-lg, font-semibold)
- 차트 높이: 300px
- 데이터: 전체 일지의 도구별 사용 횟수 합산, 상위 10개
- X축: 사용 횟수 (하단)
- Y축: 도구 이름 (좌측, text-sm)
- 바 색상: `--chart-3` (주황 계열)
- 바 호버: opacity 0.8, 툴팁에 도구명 + 사용 횟수 표시
- 바 border-radius: 우측만 4px
- 바 높이: 24px, 바 간격: 8px


#### 우측: 작업 유형 분포 (Donut Chart)

- 각 조각 색상: `--chart-1` ~ `--chart-5` 순환
- 호버: 해당 조각 확대(scale 1.05), 툴팁에 타입명 + 건수 + 비율(%)
- 범례: 차트 하단, 가로 배치(flex-wrap), 색상 원(8px) + 타입명(text-xs) + 비율
- 범례가 3개 이상일 때 줄바꿈 허용

### 섹션 1-4: 기간별 트렌드 차트
- 역할: 시간에 따른 활동 추이 파악
- 레이아웃: 전체 너비 1열, 카드 컨테이너
- 카드 타이틀: "일별 활동 추이" (TrendingUp 아이콘 + text-lg, font-semibold)
- 차트 높이: 250px
- 차트 형태: Area Chart (2개 영역)
- X축: 날짜 (MM-DD 형식), text-xs
- Y축 좌측: 세션 수, 라벨 text-xs
- Y축 우측: 도구 호출 수, 라벨 text-xs
- 영역 1 (세션 수): `--chart-1` (파란 계열), fill opacity 0.15, stroke 2px
- 영역 2 (도구 호출 수): `--chart-3` (주황 계열), fill opacity 0.15, stroke 2px
- 호버: 수직 가이드라인 + 툴팁 (날짜, 세션 수, 도구 호출 수, 일지 수)
- 범례: 차트 상단 우측, 가로 배치 (색상 선 + 라벨)
- 날짜에 여러 일지가 있을 경우: 합산하여 하나의 데이터 포인트로 표시
- 데이터 포인트 클릭:

  - 해당 날짜에 일지 여러 개: 해당 날짜의 일지 목록 팝오버 표시 (목록 페이지 달력 뷰와 동일한 팝오버)

### 섹션 1-5: 기술 스택 빈도

### 섹션 1-6: 최근 활동 목록
- 역할: 최근 일지로의 빠른 접근 경로
- 레이아웃: 전체 너비 1열
- 카드 타이틀: "최근 활동" (Clock 아이콘 + text-lg, font-semibold)
- 표시 개수: 최근 7일 (데이터가 있는 날짜 기준), 같은 날짜에 여러 일지가 있으면 각각 별도 카드

  - 일지 수 표시: 날짜 숫자 아래에 작은 텍스트 (text-xs, text-muted-foreground, "{N}건")
  - 호버: primary 색상 opacity 0.15 (라이트) / 0.25 (다크), cursor pointer
  - 클릭 동작:
    - 일지 1개: 바로 `/daily/:filename`으로 이동
    - 일지 2개 이상: 팝오버 표시

- 데이터 없는 날짜:
  - 기본 스타일, 호버 효과 없음, cursor default

- 오늘 날짜:
  - border 2px solid primary
  - 날짜 숫자: primary 색상, font-bold

- 토요일: 날짜 숫자 `--chart-1` 색상 (파란)
- 일요일: 날짜 숫자 destructive 색상 (빨간)

#### 날짜 클릭 팝오버 (일지 여러 개인 경우)
- 트리거: 데이터가 2건 이상인 날짜 셀 클릭
- 위치: 해당 셀 하단 (또는 공간에 따라 상단)
- 너비: 280px
- 스타일: 배경 `card`, border 1px solid `border`, border-radius `radius`, shadow-md, padding 12px
- 내용:
  - 헤더: "2026-02-08 (일)" (text-sm, font-semibold), margin-bottom 8px
  - 일지 목록: 세로 스택, gap 4px
    - 각 항목: padding 8px 12px, border-radius `radius`, 호버 시 accent 배경
    - 식별자 뱃지 (secondary variant) + "세션 {N}개" (text-xs, text-muted-foreground)
    - 클릭 시 `/daily/:filename`으로 이동
- 외부 클릭 시 닫힘

#### 달력 뷰 툴팁
- 데이터 있는 날짜 호버 시 (클릭 전):
  - 지연 시간: 500ms
  - 내용: "일지: {N}개 | 세션: {N}개 | 도구 호출: {N}회"
  - 스타일: 배경 `foreground`, 텍스트 `background`, text-xs, padding 4px 8px, border-radius 4px

### 섹션 3-3: 리스트 뷰
- 역할: 시간순으로 일지를 스캔/탐색
- 정렬: 최신 날짜순 내림차순, 같은 날짜 내에서는 파일명 순

#### 날짜 그룹핑
- 같은 날짜의 일지는 하나의 날짜 그룹 아래에 묶음
- 날짜 그룹 헤더: text-sm, font-semibold, text-muted-foreground
  - 형식: "2026-02-08 (일)"
  - 하단 구분선: 1px solid `border`
  - padding: 8px 0
- 그룹 간 gap: 24px

#### 각 일지 카드
- border 1px solid `border`, border-radius `radius`, padding 16px, 배경 `card`
- 호버: accent 배경, cursor pointer, transition 150ms
- 카드 간 gap: 8px (같은 날짜 그룹 내)

- 우측 (flex-shrink-0):

## 사용자 흐름

### 주요 시나리오 1: 종합 현황 확인 후 특정 일지 열람
1. 사용자가 `/`에 진입 -> 통계 카드 4개와 차트 영역이 표시됨
2. 도구 사용 현황 차트에서 Read가 가장 많이 사용된 것을 확인
3. 일별 활동 추이 차트에서 2/8에 급증한 포인트에 호버 -> 툴팁에 "세션 102개, 도구 호출 1600회" 표시
4. 해당 포인트 클릭 -> 일지가 2개이므로 팝오버 표시 -> "work" 클릭
5. `/daily/2026-02-08-work` 페이지로 이동 -> 세션 상세 아코디언에서 첫 번째 세션 내용 확인

### 주요 시나리오 2: 달력에서 일지 탐색
1. 사용자가 헤더 "Daily Logs" 클릭 -> `/daily`로 이동
2. 달력 뷰(기본)가 표시됨, 현재 월의 데이터 있는 날짜가 하이라이트
3. 2월 8일 셀에 "2건" 표시 확인 -> 클릭
4. 팝오버에 "work", "side-project" 2개 항목 표시
5. "work" 클릭 -> `/daily/2026-02-08-work`로 이동

### 주요 시나리오 3: 리스트 뷰에서 일지 탐색
1. `/daily`에서 우측 상단 뷰 토글의 리스트 아이콘 클릭
2. 날짜별로 그룹핑된 카드 목록 표시
3. "2026-02-08" 그룹 아래 "work" 카드 클릭
4. `/daily/2026-02-08-work`로 이동

### 주요 시나리오 4: 일지 간 순차 탐색
1. `/daily/2026-02-08-work` 상세 페이지에서 내용 확인
2. 우측 `>` 버튼 클릭 -> 다음 일지(index.json 순서 기준)로 이동
3. 좌측 `<` 버튼 클릭 -> 이전 일지로 복귀
4. "목록" 링크 클릭 -> `/daily`로 이동

### 대안 시나리오: 다크 모드 전환
1. 우측 상단 다크 모드 토글(Sun 아이콘) 클릭
2. 즉시 다크 모드 적용 (Moon 아이콘으로 변경)
3. localStorage에 설정 저장, 새로고침 후에도 유지

| 일지 데이터 1개뿐 (트렌드 차트) | "데이터가 충분하지 않아 추이를 표시할 수 없습니다." 메시지 + 단일 포인트 표시 |
| 기술 스택 20개 초과 (단일 카테고리) | 초기 15개 + "더 보기" 버튼 |
| 특정 섹션 데이터 누락 (마크다운 파싱 실패) | 해당 섹션 카드 자체를 렌더링하지 않음 (빈 카드 표시 안함) |
| 매우 큰 숫자 (10000+) | 천 단위 콤마 포맷 (예: 1,600) |
| 작업 유형이 매핑 테이블에 없는 경우 | 기본 아이콘으로 CircleDot (lucide-react) 사용 |
| index.json의 파일이 1개이고 상세 페이지 진입 | 이전/다음 버튼 모두 disabled |
| 도구 사용 횟수가 모두 동일한 경우 | 바 차트에서 모든 바가 동일 길이, 정상 표시 |
| 달력에서 데이터가 없는 월로 이동 | 빈 달력 표시 (데이터 하이라이트 없음), 메시지 없음 |
| 브라우저 뒤로가기 | React Router 히스토리 기반 정상 동작 |

## 접근성 요구사항

### 시맨틱 구조
- `<header>`: 헤더 영역 (로고, 네비게이션, 다크 모드 토글)
- `<nav>`: 헤더 내 네비게이션 링크 그룹
- `<main>`: 페이지별 메인 콘텐츠 영역
- `<section>`: 각 카드 섹션 (통계 카드 그리드, 차트 영역, 최근 활동 등)
- `<article>`: 일지 카드, 활동 카드 개별 항목
- `<table>`, `<thead>`, `<tbody>`, `<th>`, `<td>`: 도구 통계 테이블, 프로젝트 세션 테이블

### aria 속성
- 헤더 네비게이션: `aria-label="메인 네비게이션"`
- 현재 페이지 링크: `aria-current="page"`
- 다크 모드 토글: `aria-label="다크 모드 전환"`, `aria-pressed="true/false"`
- 뷰 전환 토글 그룹: `aria-label="보기 방식 선택"`, 각 버튼에 `aria-label="달력 보기"` / `aria-label="목록 보기"`
- 통계 카드: `aria-label="{라벨}: {값}"` (스크린 리더에서 카드 내용을 한번에 읽도록)
- 차트 영역: `aria-label="{차트 타이틀}"`, `role="img"` + 대체 설명 텍스트(sr-only)
- 아코디언: `aria-expanded="true/false"`, `aria-controls="{패널 ID}"`
- 이전/다음 네비게이션: `aria-label="이전 일지"` / `aria-label="다음 일지"`, disabled 시 `aria-disabled="true"`
- 달력: `role="grid"`, `aria-label="일지 달력"`, 각 셀 `role="gridcell"`
- 팝오버: `role="dialog"`, `aria-label="일지 목록"`
- 에러 Alert: `role="alert"`
- 로딩 스켈레톤: `aria-busy="true"`, `aria-label="로딩 중"`

### 키보드 네비게이션
- Tab 순서 (홈):
  1. 헤더: 서비스명 링크 -> Dashboard 링크 -> Daily Logs 링크 -> 다크 모드 토글
  2. 최근 활동 카드들 (각 카드에 Tab 진입 가능) -> 전체 보기 링크
  3. 차트 영역은 Tab으로 건너뜀 (정보 표시 전용)

- Tab 순서 (일지 상세):
  1. 헤더 (공통)
  2. "목록" 링크 -> 이전 버튼 -> 다음 버튼
  3. 세션 아코디언 트리거들 (순차 Tab)
  4. 아코디언 트리거에서 Enter/Space: 펼침/접힘 전환

- Tab 순서 (일지 목록):
  1. 헤더 (공통)
  2. 뷰 전환 토글 (좌/우 방향키로 전환)
  3. 달력 뷰: 월 네비게이션 화살표 -> 데이터 있는 날짜 셀 (방향키로 셀 간 이동, Enter로 선택)
  4. 리스트 뷰: 각 카드 (Tab으로 순차, Enter로 선택)

- Escape 키:
  - 팝오버 열린 상태 -> 팝오버 닫힘, 포커스 원래 셀로 복귀

### 포커스 관리
- 페이지 이동 시: 메인 콘텐츠 영역 상단으로 포커스 이동
- 팝오버 열림 시: 첫 번째 항목으로 포커스 이동
- 팝오버 닫힘 시: 트리거 요소로 포커스 복귀
- 아코디언 펼침 시: 콘텐츠 영역으로 포커스 이동하지 않음 (트리거에 유지)

### 색상 대비
- 모든 텍스트: WCAG AA 기준 (4.5:1) 이상
- foreground on background: 라이트/다크 모두 충족 (shadcn 기본 토큰 기준)
- muted-foreground on background: 확인 필요 -- 최소 4.5:1 보장
- 차트 색상: 색상만으로 구분하지 않고, 범례 라벨 + 툴팁으로 보조 정보 제공

### 터치 타겟
- 모든 클릭 가능 요소: 최소 44x44px 터치 영역
  - 헤더 네비게이션 링크: 패딩으로 44px 높이 확보
  - 달력 날짜 셀: 최소 48px (모바일)
  - 이전/다음 버튼: 36x36px 시각적 크기, 44x44px 터치 영역 (패딩)
  - 아코디언 트리거: 전체 행 클릭 가능 (높이 최소 48px)

### prefers-reduced-motion
- `prefers-reduced-motion: reduce` 시:
  - 스켈레톤 animate-pulse 비활성화
  - 아코디언 펼침/접힘 애니메이션 비활성화 (즉시 전환)
  - 차트 전환 애니메이션 비활성화
  - 호버 transition 비활성화

## 기획서 대비 확인 사항

아래는 기획서에 명시되지 않았거나 모호하여 확인이 필요한 사항입니다:

1. **달력 뷰 시작 요일**: 일요일 시작으로 설계했으나, 월요일 시작이 필요한지?
   - 현재 가정: 일요일 시작

2. **트렌드 차트 날짜 클릭 시 여러 일지 팝오버**: 기획서에는 "해당 날짜의 `/daily/:filename` 페이지로 이동"으로만 기술. 하루에 여러 일지가 있을 수 있으므로 달력 뷰와 동일한 팝오버 패턴을 적용했으나, 다른 방식이 필요한지?
   - 현재 가정: 1개면 바로 이동, 여러 개면 팝오버

3. **리스트 뷰 페이지네이션**: 일지가 매우 많아질 경우(50개+) 무한 스크롤 또는 페이지네이션이 필요한지?
   - 현재 가정: 전체 표시 (정적 데이터이므로 양이 제한적일 것으로 예상)

4. **홈 대시보드 최근 활동 "최근 7일"의 기준**: 데이터가 있는 날짜 기준 7일인지, 캘린더 기준 최근 7일인지?
   - 현재 가정: 데이터가 있는 날짜 기준 최근 7일치

5. **다크 모드 토글 3단계(light/dark/system) vs 2단계(light/dark)**: 기획서에 system 포함 3단계로 기술. 토글 버튼 UI를 2단계(Sun/Moon 클릭 전환)로 설계했으나, 3단계 드롭다운이 필요한지?
   - 현재 가정: 2단계 토글 (초기값은 system 감지, 이후 수동 전환)
